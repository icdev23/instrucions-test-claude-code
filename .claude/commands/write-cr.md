Você deve executar a tarefa de **Criar Change Request (CR)**.

**Instruções**:
1. Leia o arquivo `instructions/tasks/task_write_cr.md` para entender o fluxo completo da tarefa
2. Leia o arquivo `instructions/context/context_task.md` para entender os tipos de inputs disponíveis
3. Leia o arquivo `instructions/context/context_project.md` para obter contexto do projeto atual
4. Execute a tarefa conforme as instruções específicas do arquivo de tarefa
5. Consulte o template em `instructions/templates/template_CR.md`
6. Salve o output em `outputs/crs/` com nome descritivo

**Importante**:
- Você deve listar as funcionalidades alteradas e confirmar com o utilizador antes de escrever os CRs
- Pergunte se deve criar os CRs no Azure DevOps após a escrita
