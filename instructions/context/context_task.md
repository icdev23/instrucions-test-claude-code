* **Especificação Funcional (parcial)**  
  * Descrição  
    * Parte da *Especificação Funcional* atual da aplicação, onde estão especificados as funcionalidades e requisitos que estão sujeitos a alteração ou evolução no contexto de uma tarefa específica.  
  * Como usar  
    * Deve ser fornecido pelo utilizador.

* **Solicitação e Requisitos iniciais**  
  * Descrição  
    * Detalhamento da solicitação ou demanda, que pode ser apresentada em diversos formatos (como e-mail do cliente, *issue* do DevOps, documento externo ou fornecida diretamente no *prompt*). Contém as informações essenciais sobre os requisitos e junto com as Notas de Reunião, serve como base para a elaboração da especificação de requisitos e artefatos das funcionalidades a serem desenvolvidas.   
  * Quando usar  
    * Quando fornecido ou mencionado pelo utilizador.  
  * Como usar  
    * Se fornecido apenas um ID, titulo o link do issue, utilizar o MCP Azure DevOps disponível para consultar os work item do projeto.  
    * Confirme com o utilizador o nome do projeto correto.

* **Notas de Reunião**  
  * Descrição  
    * Notas e transcrições de reuniões. Elas contêm detalhes de informações essenciais, como contexto, definições e questões, relacionadas aos novos requisitos. Deve ser utilizada em conjunto com a Solicitação e Requisitos iniciais como base para a elaboração da especificação e artefatos.  
  * Quando usar  
    * Quando fornecido ou mencionado pelo utilizador.

* **Figma**  
  * Descrição  
    * Design da funcionalidade em Figma para dar contexto e suporte visual.  
  * Quando usar  
    * Quando fornecido ou mencionado pelo utilizador.  
  * Como usar  
    * Se fornecido print em prompt, usar os print,   
    * Se fornecido em formato de link use o servidor MCP Figma disponível.
